# Sacramento — функционал системы

Что умеет каждая часть, где какие данные живут и какие внешние сервисы используются.
Техническое «как запустить/задеплоить» — в [РУКОВОДСТВО.md](РУКОВОДСТВО.md).

---

## 1. Витрина (sacramento.md)

| Функция | Как работает |
|---|---|
| Подбор по автомобилю | На главной: марка → модель → год (данные из нашего справочника). Выбранное авто закрепляется бейджем в шапке, каталог фильтруется |
| Поиск | По названию, артикулу и OEM-номерам; терпит опечатки, пробелы и дефисы («8K0 121-251» найдёт «8K0121251H») |
| Каталог | Фильтры: категория, «только в наличии»; сортировка по цене; бейдж «✓ Подходит для вашего авто» |
| Карточка товара | Фото, цена в MDL, наличие, OEM-номера с копированием в клик, список применимости, описание |
| Корзина | Без регистрации, хранится в браузере покупателя |
| Оформление | 4 шага: контакты → получение (курьер 50 MDL / самовывоз) → оплата → комментарий. Оплата офлайн: наличные курьеру либо карта/наличные на точке |
| «Заказ принят» | Номер заказа + что дальше (менеджер позвонит за 15–30 мин) |

Личного кабинета нет — заказ по имени и телефону, как договаривались.

## 2. Админка (admin.sacramento.md)

| Экран | Что умеет |
|---|---|
| Сводка | Новые заказы (бейдж обновляется каждые 30 сек), заказы за сегодня, товары с нулевым остатком, текущий курс |
| Товары | Таблица с поиском/фильтрами; вкладки редактора: основное (+OEM-номера), цены (живой расчёт по курсу при вводе закупки), фото, применимость, склад (остаток/резерв/полка) |
| Заказы | Фильтр по статусам; в карточке: смена статуса (подтверждение резервирует остатки), переключатель розница/опт, ручная скидка (% или сумма), причина отмены |
| Курс и цены | НБМ / вручную, глобальная наценка %, правило округления, история курса |
| Импорт/экспорт | История ежедневных снэпшотов (CSV/XLSX, скачать), «Выгрузить сейчас», импорт с предпросмотром «Создаётся N · Обновляется M · Ошибки K», разовый импорт учётного .xls |
| Применимость | **Распознавание VIN** (см. раздел 4), справочник автомобилей, очередь «нераспознанные товары» с быстрой привязкой |
| Настройки | Доставка, адрес/часы самовывоза, лимит фото, глубина снэпшотов, смена пароля |

Уведомления о заказах — внутри админки (счётчик), без SMS/e-mail — по договорённости.

## 3. Цены и курс

- Закупочная цена хранится в валюте закупки (USD / EUR / MDL).
- **Розница = закупка × курс × (1 + наценка%) → округление вверх (до 1 или 5 леев).**
- Курс тянется с **bnm.md** (Нацбанк Молдовы, официальный XML, бесплатно) трижды в день: 07:00 / 10:00 / 13:00. Режим «вручную» — курс задаёт админ.
- Наценка каскадом: своя у товара → своя у категории → глобальная.
- Любая смена курса/наценки/округления автоматически пересчитывает все цены, кроме помеченных как «ручная цена».
- Опт — отдельное поле, на витрине не показывается; применяется админом к заказу.

## 4. VIN и совместимость — где наши данные, а где внешние ⚠️ важно понимать

Это два разных механизма:

### Поиск совместимости («какие запчасти подходят к авто») — ПОЛНОСТЬЮ НАШ, без внешних запросов

Источник — **наша база применимости** в PostgreSQL:
- `vehicles` — справочник автомобилей (марка, модель, годы);
- `product_vehicles` — связки «запчасть ↔ автомобиль».

Заполняется она: (а) автоматически при импорте вашего учётного .xls — парсер вытаскивает
применимость из названий товаров («Audi A4 8/97>01 радиатор…» → Audi A4 1997–2001),
(б) вручную в админке. **Качество подбора на сайте = качество этой базы.**
Никакой внешний сервис в подборе не участвует — бесплатно, мгновенно, не зависит от чужого аптайма.

### Распознавание VIN (админка, экран «Применимость») — три уровня

| Уровень | Что даёт | Куда ходит | Цена |
|---|---|---|---|
| 1. Локальный декодер | марка (по WMI — первым 3 символам VIN, словарь ~120 производителей в нашей БД) + модельный год (10-й символ VIN) | никуда | 0 |
| 2. vPIC (vpic.nhtsa.dot.gov) | иногда модель/двигатель | API правительства США | бесплатно, без ключей и лимитов |
| 3. Наш справочник | кандидаты: автомобили той же марки с подходящим годом | наша БД | 0 |

Нюансы, о которых стоит помнить:
- **vPIC заточен под рынок США** — для европейских VIN модель определяется не всегда.
  Поэтому он только «обогащение»: упал или промолчал (таймаут 3 сек) — работаем
  маркой+годом из локального декодера, ничего не ломается.
- Каждый VIN кэшируется (`vin_cache`) — повторное распознавание внешних запросов не делает.
- Rate-limit: 10 VIN-запросов в минуту с одного IP (защита от перебора).
- Точность «до комплектации» бесплатно не даёт никто — такие данные продают
  Laximo/TecDoc по лицензии. Если понадобится — платный декодер подключается
  как ещё один адаптер в существующую цепочку, без переписывания.

### Сводно: все внешние зависимости системы

| Сервис | Зачем | Стоимость | Поведение при сбое |
|---|---|---|---|
| vpic.nhtsa.dot.gov | модель по VIN (обогащение) | бесплатно | пропускается, работаем своими данными |
| bnm.md | официальный курс USD/EUR | бесплатно | действует последний известный или ручной курс |

Больше система не зависит ни от чего внешнего.

## 5. Заказы и склад

- Статусы: Новый → Подтверждён → Сборка → Доставка / Готов к выдаче → Завершён; отмена возможна на любом этапе с указанием причины.
- **Резервирование остатков — в момент подтверждения админом**, не при оформлении на сайте. Если на складе не хватает (зарезервировано другим заказом) — система не даст подтвердить и скажет, сколько доступно.
- «Завершён» — списание с физического остатка; отмена подтверждённого — возврат резерва.
- Цены в заказе фиксируются при оформлении: последующие изменения курса заказ не меняют.
- Аудит-лог: кто и когда менял курс, наценку, статусы, проводил импорт.

## 6. Данные: снэпшоты, импорт, бэкапы

- **23:55 ежедневно** — снэпшот всего каталога (артикул, название, бренд, категория, OEM, закупка+валюта, наценка, розница, опт, остаток, полка, применимость, активность) в CSV и XLSX; хранятся последние 30, скачиваются в админке.
- **Импорт** того же формата: предпросмотр (что создастся/обновится, ошибки по строкам) → подтверждение → запись. Цикл «экспорт → импорт» без потерь — формат симметричный.
- **Учётный .xls** («наличие») — отдельный разовый импорт со своим маппингом колонок, парсингом применимости из названий, полок «43*19» и обработкой дублей артикулов.
- На сервере дополнительно: ежедневный `pg_dump` (14 дней хранения).

## 8. Безопасность

- Админка — только под паролем (bcrypt, сессии, CSRF-защита); задержка при неверном пароле против перебора.
- Rate-limit на VIN (10/мин) и оформление заказа (5/мин) с одного IP.
- Закупочные цены, опт, полки, заметки — никогда не попадают в публичное API: у витрины и админки разные DTO, это закреплено тестом.
- HTTPS на всех доменах (Let's Encrypt), фото и файлы проверяются по типу и размеру.
